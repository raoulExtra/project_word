# Core Concepts Glossary

Fundamental concepts that serve as building blocks for knowledge representation, reasoning, and communication across all disciplines.

## Terms

### c/ - Concepts
- [Concept](c/def_concept.md) - Mental construct serving as building block for knowledge
- [Configuration](c/def_configuration.md) - Arrangement defining how components are organized
- [Complexity](c/asp_complexity.md) - Degree of interconnection and emergent behavior
- [Conceptual](c/asp_conceptual.md) - Mental representations and knowledge organization

### d/ - Distinctions
- [Distinction](d/def_distinction.md) - Marking a difference or difference that is acknowledged

### e/ - Entities
- [Entity](e/def_entity.md) - Distinct, identifiable object or being
- [Essential](e/asp_essential.md) - Fundamental or indispensable characteristics

### g/ - Guides
- [Core Concepts Guide](g/def_core_concepts_guide.md) - Guide covering fundamental concepts for knowledge representation

### m/ - Measures
- [Measure](m/def_measure.md) - Quantity obtained by measuring

### q/ - Quantities
- [Quantity](q/def_quantity.md) - Property that can be measured

### r/ - Relations
- [Relation](r/def_relation.md) - Way entities are connected
- [Relational](r/asp_relational.md) - Concerning the relationships between entities

### s/ - States & Systems
- [System](s/def_system.md) - Set of interrelated components working together
- [State](s/state/def_state.md) - Condition of a system at a particular point in time
- [Behaviour](s/behaviour/asp_behaviour.md) - Observable actions or responses of a system
- [Function](s/function/asp_function.md) - Purpose or role that a system serves
- [Structure](s/structure/asp_structure.md) - Arrangement of components and their relationships
- [Configuration](c/def_configuration.md) - Arrangement defining how components are organized
