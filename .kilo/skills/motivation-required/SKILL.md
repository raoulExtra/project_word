# Change Motivation Required

All code changes must include a clear, explicit motivation explaining:

- **Why** the change is needed (problem, requirement, or goal)
- **What** will be changed (scope, files affected)
- **How** it addresses the need (approach)

## Format

```
## Change: [Brief title]

**Motivation**: [Problem being solved or requirement]

**Files**: [List of affected files]

**Approach**: [How this change addresses the need]
```
