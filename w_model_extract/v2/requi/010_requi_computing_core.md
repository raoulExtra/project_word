computing needs a sequence of word introduction
Computing is not a flat domain. It’s a dependency‑layered conceptual system.

You cannot meaningfully introduce:

“compiler” before “program”

“program” before “code”

“code” before “instruction”

“instruction” before “machine”

“algorithm” before “procedure”

“variable” before “value”

“function” before “input/output”

Computing concepts depend on each other.
So a sequence is not just pedagogical — it’s structural.
